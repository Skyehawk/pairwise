Introduction

This help page covers a variety of topics in data collection and strategies to get the most from your collected data for use with the Pairwise tool to perform singular value decomposition (SVD) rank recovery (i.e., generate picklists from comparing teams against each other). There are some limitations and strategies the end user can employ to ensure a smooth user experience, limit potential issues, and ultimately achieve a greater success rate.

First and foremost - you are NOT required to use this data collection tool for generating the comparisons. You can do this in your own app, by hand (surprisingly easy), or directly into a .csv file. This tool is just provided as an easy to access (and hopefullly easy to use) service, for exploration and understanding of the SVD process and as a readily accessible option.

Pairwise data collection is meant to be lightweight and NOT intended to function as a data storage application. The data collection tool can best be thought of as a calculator, where the end user is responsible for saving the output to a file after performing the match observations. This is designed to function in as streamlined a process as possible.

Consistency is the key for success - With experience comparing teams to one another you may become more nuanced in your approach. This is an important skill, however it is vitally important that the manner in which you rank teams stays consistent throughout data collection throughout the event, a rubric may help in this endevor. Being consistent may prove to be difficult when there are many categories to place the alliance members relative to eachother and there are several people performing the comparisons. Limiting the number of categories and discussing if your ranking for that alliance in that match is consistent with how you approached previous matches may improve your consistency.

Underpinning Concepts

Levels of data - N.O.I.R

Any set of data can be placed into one of four categories, Nominal, Ordinal, Interval, or Ratio. These categories also known by the initialism NOIR represent the four distinct measurement scales that are used in statistics (or used for anything for that matter).

  • Nominal: This is the simplest level of measurement and is just used in the grouping of items that have no distinct order.
    • Robot drivetrains - each robot has a type of drivetrain, how well that drivetrain performs would be a different metric
    • Championship division names - different but with no meaningful order
  • Ordinal: There is an order or ranking to the data but it cannot be quantified between observations or even within the same observation.
    • Seed number of an alliance captain - it doesn't matter how much the first seeded team gets the position by, the second seeded team will still be next no matter what
    • Award categories - some awards may be more prestigious than others, but that is not quantifiable. The Impact Award will be seen as more prestigious than a website award but the "By how much" is not necessarily answerable
  • Interval: There is an order to the measurements and an equal distance between the measurements. However, there is no absolute zero to the scale. Measurements are comparable between themselves but not to a fundamental zero point.
    • Match scores and performance - values are comparable between matches, but the lack of a score doesn't imply a lack of performance by a robot (eg. 2017 gear bracketed scoring)
    • Time taken for tasks - The time taken by a team to complete a certain task in the competition is measurable in minutes and seconds. The time intervals are consistent, but zero doesn't signify the absence of time.
    • Gear Ratio Values - the actual numerical gear ratio values (e.g., 2:1, 3:1, etc.) These ratios demonstrate the quantitative relationship between the number of teeth or size of gears and help determine how much one gear rotates concerning another, influencing speed, torque, and power transmission. However, there is no "zero"; a 2:1 reduction can be performed with a 20-tooth and a 10-tooth gearset or a 200-tooth and 100-tooth gearset
  • Ratio: The highest level of measurement. Data here can be ranked has equal distance between intervals and is relative to a meaningful zero point.
    • Mass of Robot: If students were required to measure the mass of their robot. A measured mass of zero implies the absence of mass and therefore no robot ðŸ˜¢ .
    • Number of Points Earned: Total points earned by a team throughout the competition. This data has a true zero point, meaning a score of zero indicates an absence of points.
Pairwise SVD can operate with any Ordinal or Interval type data.

Getting Started

The Pairwise Data Input & Formatter Tool is designed to be as user friendly as possible without losing too much fine-grain control that may be desired with experience.

Simple use - Ease of understanding for consistency
In its most basic configuration the data input tool provides a "Tier-maker" style input where (considering a match completly in isolation from the rest of the event or other teams competing that season) the user ranks the teams. This allows for basic "Greater-Than", "Less-Than" , and "Equal-To" style comparisons within the alliance (intra-alliance comparisons; ordinal level data). Advanced use - More granular control for separability
The main controls of the Advanced mode Pairwise Data Input & Formatter Tool are the six sliders; three for red, three for blue. These are used to generate the output that is ultimatally used by the The SVD File Processing Tool to return the rankings. Additional information on the Advanced Mode, which supports weighted observations, will be coming soon.

The SVD processing tool is not limited to using the input tools provided within Pairwise. The output of the tools is in a generally human-readable format whtat should not be overly difficult to emulate provided the data are collectd in the alliance - relative manner. This also opens up oppertunities for inter-alliance comparison or even inter-match comparisons. This raises the opertunity to selectivly rank teams that play against eachother or in back-to-back matches to improve seperability when the rank recovery is conducted with the SVD File Processing Tool.

FAQs & Troubleshooting

FAQs

FAQ 1: Touch support doesn't work, I need to use the data input and formatter tool offline on a computer with a mouse for an event. What can I do?
Answer 1: Short of connecting a mouse to an android device, the data input and formatter tool runs locally, so as long as the computer browser load the .html you will be fine. On Firefox the can be accomnvlvplished in offline mode by:

  1. While connected to the internet, in a new browser window, open the Data Input andd Formatter Tool
  2. Go to "File" in the upper left (hit "Alt" to show "File" if it is not there)
  3. Toggle the "Work Offline" checkbox
This will unfortunatally mean any other tabs in Firefox are not connected as all of Firefox on that machine is now in offline mode.

FAQ 2: The Blue Alliance event code/match number for my upcoming event is not returning any teams. What could be the reason?
Answer 2: This integration requires an internet connection which may work well on mobile devices, but not on laptops. If you are testing before the event and connected to the internet this is likely down to the schedule not being released yet, test with a code from a previous year instead. you can always manually input the team numbers into the text boxes, just be careful that the team numbers oare correct.
